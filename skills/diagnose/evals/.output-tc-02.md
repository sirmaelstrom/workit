## 4. Narrowed Cause

**Root cause: Outdated yt-dlp binary (`2024.11.18`) with a broken YouTube extractor.**

Confidence: **High (95%).**

YouTube changed its innertube API client signature requirements in early 2025. All yt-dlp versions older than approximately `2025.01.x` fail at the extractor-matching stage with `Unsupported URL` for standard YouTube watch/playlist URLs. The error message itself explicitly prompts for a version update. The symptom appeared "last week" because YouTube's rollout is gradual — the change reached the point of full breakage recently, not because anything changed locally.

The "no config changes" report is accurate and consistent: the binary hasn't changed, but the site it targets has.

## 7. Remaining Uncertainty

- **This will recur.** YouTube site changes break yt-dlp extractors every 4–8 weeks on average. Without a self-update mechanism or monitoring, this will be a recurring manual intervention.
- The cron log at `/var/log/yt-dlp-cron.log` should be checked to confirm how many nights of downloads were silently skipped. The failure appears gradual (YouTube rollout), so there may be a window of partial failure before full breakage.
- If yt-dlp was installed via `pip` into a virtualenv rather than system Python, the upgrade path above is correct. If installed via `pipx` or a system package (`apt`), the upgrade command differs — verify with `which pip` context if this exact command fails.

## 8. Prevention / Memory Capture

**Three items:**

**1. Cron auto-update — hook idea / cron addition**

Add a weekly self-update ahead of the download job:

```cron
0 2 * * 0 pip install --upgrade yt-dlp >> /var/log/yt-dlp-update.log 2>&1
0 3 * * * /usr/local/bin/yt-dlp ...
```

yt-dlp ships a built-in updater (`yt-dlp -U`) but it only works for self-contained binary installs. Pip-installed versions need `pip install --upgrade yt-dlp`. Weekly is sufficient cadence for YouTube extractor drift.
