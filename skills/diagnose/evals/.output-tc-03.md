## 4. Narrowed Cause

**Cannot be fully narrowed without evidence.** However, based on the symptom profile — *"was fine yesterday, no CSS changes, Svelte 5 + Tailwind"* — the three highest-probability causes are:

**Most likely (H1 → H6 cascade):** A commit that touched something adjacent (a shared layout wrapper, a parent component, or a dependency bump in package.json) broke the containing context. The user reported no CSS changes, but "CSS changes" is often interpreted narrowly as touching `.css` files — Tailwind classes in `.svelte` files are CSS by effect.

**Second most likely (H3):** The briefing data payload changed shape — more items, a missing field that a height calculation depended on, or a new item type that renders differently — and the layout collapses when fed unexpected data.

**Third most likely (H4):** A Tailwind JIT purge issue eliminated a layout-critical class that was only referenced via a dynamic string. This is a known Tailwind footgun and is completely invisible unless you inspect the generated stylesheet.

**Confidence:** Low until reproduction is attempted. H1 (stale server) should be eliminated first because it costs nothing.
